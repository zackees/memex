How it works

This page uses sqlite-wasm-http running in a single background Web Worker. The database is not downloaded — only the SQLite pages needed per query are fetched via HTTP range requests (typically <1% of the total DB size).

The WASM binary is optimized with wasm-opt -Oz (1.45 MB, ~515 KB gzip).

Schema 
CREATE TABLE chunks (
    id INTEGER PRIMARY KEY, source_type TEXT, path TEXT,
    title TEXT, body TEXT, metadata TEXT
);
CREATE VIRTUAL TABLE search_trigram USING fts5(
    source_type, path, title, body, metadata,
    content=chunks, content_rowid=id, tokenize='trigram'
);
CREATE VIRTUAL TABLE search_porter USING fts5(
    source_type, path, title, body, metadata,
    content=chunks, content_rowid=id, tokenize='porter unicode61'
);
